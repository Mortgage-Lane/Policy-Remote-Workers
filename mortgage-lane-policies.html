Setting Up MFA for Microsoft 365

Multi-factor authentication protects your account from unauthorised access. Follow the steps below to get set up on your mobile device.

Security IT
1

Install Microsoft Authenticator

▾

Download the Microsoft Authenticator app on your phone before starting setup.

  • iPhone: search Microsoft Authenticator in the App Store
  • Android: search Microsoft Authenticator in the Google Play Store
2

Trigger the MFA Setup

▾
  • Sign into your Microsoft 365 account at microsoft365.com
  • You will be prompted to set up MFA — or navigate directly to aka.ms/mfasetup
  • Click Next when asked to provide additional security information
3

Add Your Account in the App

▾
  • Choose Microsoft Authenticator app as your method
  • On your phone, open the app, tap +, then select Work or school account
  • Scan the QR code displayed on your computer screen
  • Approve the test notification sent to your phone to confirm setup
4

Set Authenticator as Your Default

▾

In your Microsoft security settings, ensure Authenticator is set as your default sign-in method for the best experience.

Back up your Authenticator account in the app settings — this makes it easy to restore if you ever change your phone.
✦

Using a Third-Party Authenticator App

▾

If you prefer to use a third-party TOTP app such as Google Authenticator, Authy, or 1Password, Microsoft 365 supports this via a one-time code (TOTP) method. Follow the steps below after completing Step 2 above.

Step A — Choose the TOTP option

  • At the MFA setup screen, click I want to use a different authenticator app
  • Click Next — Microsoft will display a QR code

Step B — Scan the QR code in your app

  • Open your chosen authenticator app on your phone
  • Tap + or Add account (wording varies by app)
  • Select Scan QR code and point your camera at the code on screen
  • Your Microsoft 365 account will appear in the app with a 6-digit rotating code

Step C — Verify and complete setup

  • Microsoft will ask you to enter the 6-digit code currently shown in your app
  • Type the code and click Verify before it refreshes (codes rotate every 30 seconds)
  • Once verified, click Done — your account is now protected with MFA
ℹ No push notifications. Third-party apps only generate TOTP codes — you will not receive one-tap push notifications like you would with the Microsoft Authenticator app. You must open the app and type the code each time you sign in.
If your app supports cloud backup (e.g. Authy or 1Password), enable it so you don't lose access if you change phones. Google Authenticator also supports Google Account backup — make sure it is turned on in the app settings.
✦

MFA Method Comparison

▾
Method Security Convenience
Passkeys Best Excellent — FaceID, fingerprint or PIN
Microsoft Authenticator (push notification) Best One tap
Microsoft Authenticator (TOTP code) Best Good
SMS text code Weaker Good
Phone call Weaker OK
⚠ Avoid SMS where possible. SMS codes are vulnerable to SIM-swapping attacks and should only be used as a fallback.
Number matching is enabled by default in Microsoft 365 — when approving a sign-in you will enter a 2-digit number shown on your login screen, which prevents accidental approvals.
Passkeys use a cryptographic key pair stored on your device and are resistant to phishing. Where supported, they offer a stronger and simpler experience than a password and authenticator combined.

Device Hardware Specification Policy

Sets the minimum hardware standard any desktop or laptop must meet before being used for Mortgage Lane business. Applies to company-owned and personally owned work devices.

IT Security Issued 16 June 2026
✦

Purpose & Scope

▾

This policy maintains a consistent hardware baseline so that all staff devices are:

  • Fast enough for day-to-day mortgage and CRM applications
  • Capable of running a supported, fully patched operating system
  • Able to meet Cyber Essentials and FCA data-security obligations

It applies to all desktop and laptop computers used to access Mortgage Lane systems, email, or customer data — including devices used for occasional or remote work. Mobile phones and tablets are governed separately.

✦

Minimum Hardware Specification

▾

Every desktop and laptop used for work must meet or exceed all of the following:

Component Minimum requirement
Operating system A modern, vendor-supported OS (e.g. Windows 11) that receives security updates and can be enrolled in Microsoft Intune.
Processor (CPU) Intel Core i3 12th generation or higher, AMD Ryzen 3 7000 series or higher, or an equivalent processor of the same class or newer.
Memory (RAM) At least 8 GB.
Storage At least 128 GB — solid-state drive (SSD) strongly preferred.
ℹ If you are unsure whether your device qualifies, check with IT before using it for work. Windows 10 reached end of support in October 2025 and is no longer compliant.
✦

Why These Requirements?

▾
  • Supportable OS. The device must run an OS that can be enrolled in Microsoft Intune for configuration, monitoring, and remote security management.
  • Security. A supported OS and 8 GB of RAM allows disk encryption, modern anti-malware, and management tooling to run without degrading the device — supporting Cyber Essentials and FCA data-security expectations.
  • Performance. 8 GB RAM, a 128 GB+ SSD, and a current-generation processor keep everyday mortgage and CRM applications responsive and reduce downtime.
✦

Existing & New Devices

▾
  • New purchases. Any new desktop or laptop bought for work must meet this specification from the point of purchase.
  • Existing devices. Devices already in use will be checked against this standard. Any that fall short must be scheduled for replacement and should not be used to store or process customer data in the meantime.
✦

Exceptions & Review

▾

Any exception to this policy must be:

  • Approved in writing by IT or management
  • Recorded with a fixed review date
  • Accompanied by compensating controls (for example, restricted data access)
This policy is reviewed at least annually or whenever operating-system support requirements change.
